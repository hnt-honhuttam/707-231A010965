PHẦN 1: LÝ THUYẾT (40 ĐIỂM)

1. Position: absolute vs relative

  • Relative: Phần tử giữ nguyên vị trí trong luồng tài liệu. Khi dịch chuyển (top, left...), nó di chuyển so với chính vị trí gốc, không làm xô lệch phần tử khác.
  • Absolute: Phần tử bứt hoàn toàn khỏi luồng tài liệu, định vị trí dựa trên phần tử cha gần nhất có position: relative.
  • Ví dụ: Nút "X" (Đóng) ở góc Modal. Thẻ bọc Modal set relative, nút X set absolute (top: 10px; right: 10px).
  • Ảnh hưởng Deploy: Nếu lạm dụng absolute dùng px cứng, khi deploy chạy trên mobile (màn hình nhỏ), UI sẽ bị vỡ. Cần dùng Flexbox/Grid hoặc dùng % để responsive.

2. Domain, Hosting & DNS

  • Quy trình: Mua Domain -> Thuê Hosting -> Trỏ NameServer/Bản ghi A từ Domain về IP của Hosting.
  • Vai trò DNS: Hệ thống phân giải tên miền, dịch tên miền dễ nhớ thành địa chỉ IP của server để trình duyệt tìm đến.
  • Công cụ Deploy: GitHub Pages, Netlify (hỗ trợ Web tĩnh, tự động build).
  • Rủi ro bảo mật: Nếu cấu hình sai DNS dễ bị DNS Hijacking. Nếu không có SSL (HTTPS), mã JS gửi đi sẽ bị nghe lén (Sniffing).

3. Phân biệt let, const, var & Hoisting

  • Scope: var có phạm vi hàm (Function scope). let và const có phạm vi khối (Block scope {}).
  • Hoisting: Cả 3 đều bị đẩy lên đầu. var được gán undefined. Còn let/const rơi vào Vùng Chết Tạm Thời (TDZ), gây lỗi ReferenceError nếu gọi.
  • Ví dụ lỗi: console.log(a); let a = 10; (Báo lỗi Cannot access 'a' before initialization).
  • Cách Debug: Dùng Chrome DevTools (F12) -> Tab Sources -> Đặt Breakpoint và chạy Step over để kiểm tra Scope/Watch.

4. Event Delegation trong JavaScript

  • Khái niệm: Gắn một sự kiện duy nhất cho phần tử cha để quản lý các phần tử con, tận dụng cơ chế Event Bubbling (Sự kiện nổi bọt).
  • Ví dụ: Gắn sự kiện click vào thẻ <ul> thay vì gắn cho hàng ngàn thẻ <li>. Dùng e.target để kiểm tra phần tử được click.
  • Lợi ích: Xử lý tốt cho các phần tử con sinh ra động bằng JS. Tối ưu hiệu suất, giảm Memory Leak và tiết kiệm RAM trình duyệt.
